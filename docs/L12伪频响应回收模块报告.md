# L12 模块完成度报告：伪频响应回收

## 模块概述

**目标**：在检测到伪频响应时，不仅触发自毁机制，还能优雅地回收/终止正在进行的流式输出，避免用户收到部分不合规的内容，并提供明确提示。

**核心意义**：

- 保证交互体验一致性：伪频触发时不会残留部分输出。
- 增强安全性：防止伪频内容片段被误用或误解。
- 与防火墙、频率偏移检测形成闭环保护。

---

## 当前完成度

### 已完成

- **伪频检测与自毁机制**（后端 `core/detector.py`）：
  - 已能实时检测流式和非流式输出中的伪频特征。
  - 检测到后直接发送自毁提示并中止后续输出。

### 未完成

- **优雅回收策略**（核心待办）：

  - 需要在流式输出中实现「回收已发送内容」或「替换为安全占位符」。
  - 提供明确的结束标记，区分正常终止与伪频终止。

- **前端体验优化**：
  - 前端 UI 需对“回收提示”做特殊展示（如高亮、标红、加提示条）。
